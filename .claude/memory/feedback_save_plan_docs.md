---
name: Always save plan docs
description: User wants an approved plan saved as a .md file in docs/ before implementation begins
type: feedback
---

Always save a plan document as a `.md` file in the `docs/` folder before starting implementation. The plan must be user-approved first.

**Why:** The user wants a persistent record of every approved plan, not just the ephemeral plan mode file.

**How to apply:** After the user approves a plan, save it to `docs/<descriptive_name>_plan.md` before making any code changes. This applies to every plan, not just large ones.
